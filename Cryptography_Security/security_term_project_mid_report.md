# ARX 기반 스트림 암호의 멀티미디어 데이터 암호화 특성 비교

## Salsa20과 ChaCha20의 사진 및 영상 암호화 실험 설계

### 요약
본 보고서는 최종 결과 보고서가 아니라 실험을 진행하며 수정할 연구 골격이다. Salsa20과 ChaCha20의 ARX 구조 차이를 정리하고, 사진 및 영상 파일을 대상으로 암호화와 복호화 성능을 비교할 실험 절차를 설계한다. 또한 단순 처리 속도 비교를 넘어 키와 nonce 변화에 따른 키스트림 확산, 라운드 수에 따른 성능과 확산의 균형, nonce 재사용 시 멀티미디어 데이터에서 발생할 수 있는 누출을 관찰할 계획이다. 실험 결과 표와 그래프, 구현 세부 사항, 해석은 후속 실험 후 보완한다.

## Ⅰ. 서  론
스트림 암호는 대용량 데이터를 연속적으로 처리하기 쉬워 사진과 영상처럼 바이트 수가 큰 데이터의 암호화 실험에 적합하다. 본 프로젝트는 Salsa20과 ChaCha20을 직접 구현하거나 검증 가능한 구현과 대조하여 두 알고리즘의 구조 차이가 처리 성능과 키스트림 확산 특성에 어떤 차이를 만드는지 분석한다.
초기 제안서의 목표는 ARX(Addition-Rotation-XOR) 구조 분석, Salsa20과 ChaCha20의 속도 비교, avalanche effect 관찰이었다. 본 초안에서는 이를 심화하여 연구 질문을 세 가지로 정리한다. 첫째, 파일 크기와 입출력 단위가 달라질 때 암호화 및 복호화 처리량은 어떻게 달라지는가. 둘째, 키 또는 nonce의 한 비트 변화가 키스트림에 얼마나 빠르게 확산되는가. 셋째, nonce를 잘못 재사용하면 멀티미디어 데이터에서 어떤 누출이 관찰되는가.
평문 한 비트만 바꾼 뒤 ciphertext 전체의 avalanche를 측정하는 방법은 스트림 암호 해석에는 한계가 있다. 동일한 키스트림과 XOR하는 구조에서는 평문 차이가 대응 위치의 ciphertext 차이로 직접 반영되기 때문이다. 따라서 본 연구의 확산 실험은 키, nonce, 라운드 수 변화가 생성 키스트림의 Hamming distance에 미치는 영향을 중심으로 설계한다.

## Ⅱ. 관련연구
Salsa20과 ChaCha20은 덧셈, 비트 회전, XOR만으로 상태를 갱신하는 ARX 계열 스트림 암호이다. Salsa20은 quarter-round와 column/row round를 사용하며, ChaCha는 quarter-round의 연산 순서와 diagonal round 구성을 조정하여 라운드당 확산 특성을 개선하려는 변형으로 제안되었다. 두 알고리즘은 S-box 기반 구현과 다른 구현 특성을 가지므로 구조, 라운드 수, 플랫폼별 처리량을 함께 살펴볼 필요가 있다.
ChaCha20은 현대 프로토콜에서 Poly1305와 결합한 인증 암호 형태로도 사용된다. 본 프로젝트의 1차 범위는 Salsa20과 ChaCha20의 암호화 키스트림과 XOR 암호화 경로 비교이지만, 최종 보고서에서는 기밀성만 제공하는 실험과 무결성까지 다루는 ChaCha20-Poly1305의 역할 차이를 분리해 논의한다.
무작위성 검정은 생성 바이트열의 통계적 특성을 관찰하는 보조 지표로 사용한다. 엔트로피, 바이트 빈도, 선택한 NIST SP 800-22 계열 검정 결과는 구현 이상 징후를 찾는 참고 자료가 될 수 있으나, 통계 검정을 통과했다는 사실만으로 암호학적 안전성을 증명하지는 않는다.

## Ⅲ. 본론

### 3.1 연구 범위와 가설
연구 대상은 Salsa20/20과 ChaCha20을 기본으로 하며, 가능하면 Salsa20/8, Salsa20/12, ChaCha8, ChaCha12를 함께 비교하여 라운드 수 감소가 처리량과 확산 지표에 주는 영향을 관찰한다. 사진 데이터는 원시 픽셀 구조가 비교적 드러나는 BMP 또는 PPM 계열과 압축 포맷인 PNG 또는 JPEG를 나누어 사용한다. 영상 데이터는 동일 해상도와 길이의 짧은 샘플을 사용하고 파일 크기 차이가 결과를 지배하지 않도록 별도 표로 관리한다.
가설 H1: 동일한 구현 조건에서 ChaCha20은 구조 변경으로 인해 Salsa20과 비슷하거나 더 나은 처리량을 보이면서 키스트림 확산이 빠르게 나타날 것이다. 가설 H2: 라운드 수가 낮은 변형은 처리량은 높아지지만 초기 블록 또는 감소 라운드 조건의 확산 지표가 약해질 수 있다. 가설 H3: nonce 재사용 실험에서는 ciphertext 두 개의 XOR가 평문 두 개의 XOR와 연결되어 이미지 또는 영상 데이터의 구조 정보 일부가 관찰될 수 있다.

### 3.2 실험 설계와 평가 지표
성능 실험은 동일 장비, 동일 컴파일 옵션, 동일 키 길이, 동일 파일 집합에서 수행한다. 각 파일에 대해 암호화 시간, 복호화 시간, 처리량(MB/s), 반복 실행 평균과 표준편차를 기록한다. 파일 읽기와 쓰기 시간을 포함한 end-to-end 측정과 순수 암호 코어 처리 시간을 구분하면 구현 최적화와 입출력 병목을 분리할 수 있다.

정확성 검증은 알려진 테스트 벡터 또는 신뢰 가능한 참조 구현과의 결과 일치, 그리고 decrypt(encrypt(P)) = P 확인으로 진행한다. 사진은 픽셀 단위 차이가 없는지 해시로 확인하고, 영상은 원본과 복호화 파일의 해시 및 재생 가능 여부를 함께 기록한다. 암호문이 정상 디코더에서 열리지 않는 현상은 기밀성 검증의 보조 관찰일 뿐 보안성의 증명으로 해석하지 않는다.
확산 실험은 기준 키스트림과 키 한 비트 변경, nonce 한 비트 변경, counter 변화 조건의 키스트림을 생성한 뒤 블록별 Hamming distance 비율을 측정한다. 결과는 라운드 수와 출력 블록 위치별 그래프로 정리한다. 추가로 암호문 바이트 분포, Shannon entropy, 선택한 통계 검정 결과를 기록하되 확산 결과와 성능 결과를 분리해 해석한다.
오용 사례 실험은 같은 키와 같은 nonce로 서로 다른 이미지 또는 영상 바이트열을 암호화한 뒤 C1 XOR C2 = P1 XOR P2 관계를 확인하는 방향으로 진행한다. 원시 이미지 포맷, 압축 포맷, 파일 헤더를 포함한 전체 파일 암호화와 payload 중심 암호화를 나누어 관찰하면 누출이 어디서 두드러지는지 비교할 수 있다. 이 실험은 안전한 사용법을 설명하기 위한 통제된 실험으로 제한한다.

## Ⅳ. 구현

* 구현 단계 1: Salsa20과 ChaCha20의 상태 배열, quarter-round, block function, counter 증가, XOR 암호화 경로를 모듈별로 구현한다. 먼저 테스트 벡터와 참조 구현 대조로 correctness를 확인한 후 멀티미디어 파일 입출력 코드를 연결한다. 키, nonce, counter는 실험 로그에 재현 가능하게 기록하되 보고서 공개본에는 실제 비밀 키를 그대로 싣지 않는다.

* 구현 단계 2: 사진과 영상 샘플 집합을 고정하고 실험 스크립트 또는 실행 인자를 표준화한다. 파일 종류, 해상도, 프레임 수, 파일 크기, 압축 여부, 반복 횟수, 버퍼 크기, 컴파일 옵션을 표로 남긴다. [실험 후 표 1: 데이터셋 요약], [표 2: 성능 측정 결과], [그림 1: 라운드 수별 처리량], [그림 2: 키/nonce 변화에 따른 키스트림 Hamming distance]를 삽입한다.

* 구현 단계 3: nonce 재사용 실험과 정상 nonce 사용 실험을 분리한다. 정상 조건에서는 각 암호화에 고유 nonce를 사용하고 복호화 일치와 성능을 측정한다. 오용 조건에서는 동일 nonce 재사용으로 생기는 XOR 관계와 구조 누출을 시각 자료로 정리하되, 공격 가능성의 원인이 알고리즘 자체의 일반 사용이 아니라 스트림 암호 사용 규칙 위반임을 명확히 쓴다.
최종 보고서에서 보완할 항목: 실제 개발 환경과 CPU 정보, 사용한 구현의 검증 근거, 테스트 벡터 결과, 실험 반복 횟수, 통계 처리 방식, 사진과 영상 샘플 선정 근거, 한계점, ChaCha20-Poly1305를 포함할지 여부를 확정한다.

## V. 결 론 
본 초안은 Salsa20과 ChaCha20을 사진 및 영상 암호화 사례에 적용하되, 단순한 암호화 성공 화면과 속도 비교에 머무르지 않도록 연구 질문과 평가 지표를 정리하였다. 핵심 분석 축은 ARX 구조 차이, 라운드 수에 따른 성능과 확산의 균형, 키와 nonce 변화에 대한 키스트림 반응, nonce 재사용 오용이 멀티미디어 데이터에서 드러내는 누출이다.
향후 실험 후에는 실제 수치와 그래프를 근거로 가설을 검토하고, 구현 방식과 데이터셋 선택이 결과에 미친 영향을 별도로 논의한다. 최종 결론은 측정 결과가 확보된 뒤 작성하며, 현 단계의 문장은 보고서 구조와 실험 방향을 고정하기 위한 작업 초안으로 둔다.
